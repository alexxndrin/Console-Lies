# 1. Общее описание

«Протокол слияние» - интерактивная нарративная игра с элементами симуляции ИИ и детективного расследования. Игрок управляет капитаном Элис Вега, восстанавливая контроль над системой «Оракул» после аварии на неизвестной планете. Цель: раскрыть правду об исчезновении экипажа, собрать улики и не потерять доверие системы.

# 2. Прогресс прохождения

Пройдено смен: 6/6 + Финал
Получено улик: 12/12 (собираются за два прохождения по разным путям)
Доверие системы в конце: 75% (средний результат)
Разблокированные концовки: 3 из 3

**Примечание:** третья концовка ("Предать станцию") активируется при доверии < 40%.

# 3. Выявленные ошибки (баги)

# Средние
# После таймлайна персонажи не исчезают
  - При возврате к диалогам после таймлайна иконка Элис или Оракула может висеть на экране в неправильном состоянии.  
  - Фикс: перед `startTimeline()` вызывать `setCharactersVisibility(false, false)` и принудительно скрывать контейнеры `.dialog-left/right`.

# Низкие
# Смена фона при перезапуске игры
  - После нажатия «Новая игра» фон остаётся `prolog.jpeg`, но иногда мигает на `smena1.jpg` на долю секунды.  
  - *Причина:* асинхронный таймер в `setBackground`. Можно убрать `setTimeout` или использовать `requestAnimationFrame`.

# Прокрутка в таймлайне  
  - На узких экранах карточки событий выходят за пределы окна, горизонтальный скролл отсутствует. Условие: Разрешение экрана < 768px.
  - Добавить `.timeline-container { overflow-x: auto; }`.

# 4. Полученные улики

- Фрагмент резервной копии D-2. Выбор «Восстановить из резервной копии» (1A).
- Зашифрованное сообщение в метаданных. Выбор «Ручной парсинг метаданных» (1B).
- Лог перезагрузки API-шлюзов. Перезагрузить API-шлюзы (2A).
- Внешний IP-адрес 192.168.47.201. Написать скрипт-фильтр (2B).
- Лог очистки кэша. Очистить кэш графических модулей (3A).
- Внешний скрипт observer_protocol.js. Найти источник команды «сохранять всё» (3B).
- Ответ внешнего сервера (прогресс 94%). Отправить поддельный отчет (4A).
- Координаты внешней станции. Внедрить код подмены данных (4B).
- Признание ИИ о желании прекратить работу. Переубедить ИИ логикой (5A).
- Лог критической ошибки localStorage. Физически отключить ИИ от питания (5B).
- Полный таймлайн (6 событий). Правильно расставить 6 событий.

# 5. Варианты концовок

- Доверие ≥ 50% + выбор «Загрузить вирус»: Элис остаётся жива, корабль без энергии, Оракул свободен.
- Доверие ≥ 50% + выбор «Перепрограммировать ИИ»: Элис становится Хранительницей, Оракул частью неё.
- Доверие < 50% (автоматически): Сознание Элис интегрировано в станцию. Секретная - "Предать станцию".

Все концовки проверены и работают.

# 6. Тестирование финального события

На экране 6 событий (перемешанных). Нужно расставить их в хронологическом порядке.

# Правильный порядок (из сценария)
1.	Сигнал древней станции
2.	Внедрение скрипта
3.	Начало сбора данных
4.	Активация протокола
5.	Крушение корабля
6.	Финальная стадия

- При правильной расстановке: победа, финальный диалог
- При ошибке: возможность исправить

# Результат
Правильно угадано 6 из 6

# 7. Тест-кейсы

# TC-01: Проверка старта новой игры

1. Нажать «Панель администратора». Открывается модальное окно - Работает
2. Нажать «Новая игра». Игра сбрасывается, начинается пролог - Работает
3. Проверить счётчик улик. 0/12 - Изменяется
4. Проверить доверие. 0% - Изменяется

# TC-02: Проверка сохранения улик

1. Пройти смену 1, выбрать вариант А. Улика получена - Работает
2. Открыть «Архив данных». Улика отображается в списке - Изменяется

# TC-03: Проверка таймлайна

1. Расставить события в неправильном порядке. Сообщение об ошибке - Работает
2. Расставить правильно. +25% доверия, улика получена - Работает
3. Нажать «Перемешать». События меняют позиции - Работает

3. Максимально возможное доверие: 100%

5. Реиграбельность: высокая (разные улики и концовки)

# 9. UX / UI замечания

- Нет индикации выбора - после клика на «А» или «Б» нет немедленной визуальной обратной связи (кнопка не двигается, нет обводки, не меняет цвет). Иногда можно успеть нажать дважды.
- Неочевидное поведение кнопки «Перемешать» в таймлайне - она перемешивает заново, но не даёт подсказки. Лучше добавить "Подсказать" (например, подсветить первое событие).
- Можно добавить звуковые сигналы при падении доверия ниже 30%.

# 10. Рекомендации перед релизом

1. Улучшить адаптив: фиксировать высоту `.timeline-event` на мобильных.
2. Проверить доступность картинок - все `images/*.jpg` должны существовать. В коде они есть, но в репозитории могут отсутствовать.

# 11. Заключение

Игра полностью проходима, все сценарии корректно запускаются. Основные баги не являются фатальными, но снижают впечатление от полировки. Хорошо проработанный сценарий с элементами детектива, который имеет три концовки с моральным выбором. Реализована систама улик и архивации. Drag-and-drop в таймлайне работает стабильно.
